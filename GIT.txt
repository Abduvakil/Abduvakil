git clone - закачивать к себе проект
git add - добавить изменения
git commit - зафиксировать изменения
git push -  отправить изменения 
git pull - скачать проект с изменениями
смержить (merge) - обьеденить изменения одного файла из двух источников
git branch - создать аналог на основе исходного проекта
git checkout - переключить отображение варианта проекта
1. Инициализация репозитория
git init: Инициализирует новый Git-репозиторий в текущей папке.
2. Работа с файлами
git add <file>: Добавляет файл в индекс (staging area).
git add .: Добавляет все изменённые и новые файлы.
git rm <file>: Удаляет файл из рабочей директории и Git.
git mv <old_file> <new_file>: Переименовывает или перемещает файл.
3. Коммиты
git commit -m "message": Создаёт коммит с указанным сообщением.
git commit -a -m "message": Коммит всех изменённых файлов, которые отслеживаются.
4. Просмотр статуса и истории
git status: Показывает текущий статус изменений (отслеживаемые, неотслеживаемые, изменения в индексе).
git log: Показывает историю коммитов.
git log --oneline: История коммитов в одну строку.
git diff: Показывает различия между рабочей копией и последним коммитом.
git show <commit>: Показывает изменения в конкретном коммите.
5. Работа с ветками
git branch: Показывает список веток.
git branch <branch_name>: Создаёт новую ветку.
git checkout <branch_name>: Переключается на указанную ветку.
git checkout -b <branch_name>: Создаёт и переключается на новую ветку.
git merge <branch_name>: Сливает указанную ветку в текущую.
git branch -d <branch_name>: Удаляет ветку.
6. Работа с удалёнными репозиториями
git remote add origin <url>: Добавляет удалённый репозиторий.
git fetch: Загружает изменения из удалённого репозитория, но не сливает их.
git pull: Загружает и сливает изменения из удалённого репозитория.
git push origin <branch_name>: Отправляет изменения на удалённый репозиторий.
7. Отмена изменений
git reset <file>: Убирает файл из индекса (staging area).
git checkout -- <file>: Восстанавливает файл из последнего коммита.
git revert <commit>: Отменяет конкретный коммит, создавая новый коммит с отменой изменений.
8. Работа с субмодулями
git submodule add <repository>: Добавляет субмодуль.
git submodule init: Инициализирует все субмодули в репозитории.
git submodule update: Обновляет субмодули до последнего коммита.
9. Другие полезные команды
git clone <url>: Клонирует удалённый репозиторий на локальную машину.
git stash: Сохраняет текущие изменения, не создавая коммита.
git stash pop: Восстанавливает изменения, сохранённые в stash.
git tag <tag_name>: Создаёт тег на текущем коммите.
